## 二、两种方案对比

### 方案 E：从现有仓库提取（Extract）

**步骤**：
1. 把所有 Hermes 核心文件还原到原始状态（undo ~500 行修改）
2. 从仓库中删除 `hermes_cli/trade_chat.html`、`skills/b2b-document/` 等 Trade 专属文件
3. 把 `trade/` 目录和 Trade 专属文件搬到新仓库
4. 新仓库 `pyproject.toml` 中声明 `hermes-agent` 为 pip 依赖
5. Hermes 仓库继续维护（含 cognee、file_extract 等贡献）

**优点**：
- `trade/` 目录保留完整 Git 历史
- 不需要重新测试已有的 700 行 Trade 代码
- Cognee、file_extract、jina_reader 等工具已贡献给 Hermes，持续可用

**缺点**：
- 需要在当前仓库中做「手术式」剥离：undo ~500 行修改，删除 Trade 文件
- Git history 中有大量与 Trade 无关的 Hermes 提交（~400+ commits），新仓库如果全量迁移会很臃肿
- 如果剥离不干净，可能遗留 Trade 到 Hermes 的隐藏依赖

---

### 方案 N：重新 clone Hermes，重新添加 Trade（New Start）

**步骤**：
1. `git clone https://github.com/chefroger/hermes-agent` 获取干净的 Hermes
2. 从当前仓库复制以下文件到新 Trade 仓库：
   ```
   trade/          （全部 *.py）
   trade_chat.html
   skills/b2b-document/SKILL.md
   项目需求文档.md / Trade-Hermes耦合评估.md / Trade-Hermes解耦可行性.md
   ```
3. 新 Trade 仓库初始化：
   ```
   trade-ai-assistant/
   ├── pyproject.toml          # name=trade-ai-assistant, depends=hermes-agent
   ├── server.py               # 独立 FastAPI 入口
   ├── trade/                  # 复制过来的业务层
   ├── static/
   │   └── trade_chat.html     # 复制过来的 SPA
   └── skills/
       └── b2b-document/SKILL.md
   ```
4. 给 Hermes 添加 3 个最小 patch（见下方）
5. `pip install hermes-agent` → 启动 Trade

**优点**：
- 仓库完全干净，没有历史包袱
- Trade ↔ Hermes 边界一目了然，零模棱两可
- 仓库体积小（~50 files），易于理解和维护
- Git history 只包含 Trade 相关的提交

**缺点**：
- 丢掉 `trade/` 的 Git 历史（但只有 ~10 个 trade 相关的提交，损失不大）
- 需要把 cognee、file_extract、jina_reader 等工具重新贡献给 Hermes（或保留在 Trade 侧）

## 三、Trade 对 Hermes 需要的最小 patch（方案 N 需要）

如果重新 clone Hermes，Trade 不需要修改 Hermes 源码。Trade 自己启动一个 FastAPI 实例，只依赖 Hermes 的 Python API：

```python
# Trade 的 server.py — 不需要修改 Hermes
from hermes_agent import AIAgent  # pip install hermes-agent
from trade.api import router

app = FastAPI(title="Trade AI Assistant")
app.include_router(router, prefix="/api/trade")
```

但这有一个前提：Hermes 打包后需要暴露 `AIAgent` 作为公开 API。当前 Hermes 的 `pyproject.toml` 中，`run_agent.py` 是 py-modules 的一部分，安装后可以通过 `from hermes_agent import AIAgent` 导入（取决于 Hermes 打包时的包名）。

如果 Hermes 不暴露 `AIAgent` 作为公开 API（比如包名叫 `hermes_agent` 但内部模块路径不同），Trade 需要调整 import 路径，但不需要修改 Hermes 源码。

**Trade 完全不需要修改 Hermes 源码的唯一条件是**：Trade 自己启动 FastAPI，不挂载到 Hermes 的 web_server 上。

## 四、结论：推荐方案 N（重新 clone）

| 维度 | 方案 E (提取) | 方案 N (重新 clone) |
|------|-------------|-------------------|
| 工作量 | 1-2 天（手术式剥离） | 0.5-1 天（复制粘贴） |
| 代码干净度 | ⚠️ 可能遗留隐藏依赖 | ✅ 完全干净 |
| Git 历史 | ✅ 保留 | ❌ 丢失（但只有 ~10 commits） |
| 仓库体积 | ⚠️ 臃肿 | ✅ 精简 |
| 边界清晰度 | ⚠️ 依赖人工验证 | ✅ 编译期保证 |
| 长期维护 | ⚠️ 与 Hermes 的历史纠缠 | ✅ 独立演进 |

**具体操作**：

1. 保留当前仓库不动（作为 Hermes fork，维护 cognee、file_extract 等贡献）
2. 新建 `trade-ai-assistant` 仓库，只放 Trade 代码
3. 从当前仓库 `cp -r trade/ trade_chat.html skills/b2b-document/` 到新仓库
4. 新仓库 `pyproject.toml` 依赖 `hermes-agent`
5. Trade 启动自己的 FastAPI（`server.py`），不碰 Hermes 的 web_server

**预估工作量**：半天即可完成代码搬迁和验证。
