# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Foreign Trade Assistant — a B2B document Q&A application for trade and manufacturing sales teams. It's a FastAPI server that wraps the **Hermes Agent** (AI agent engine from `chefroger/hermes-agent`) with a custom B2B business layer (document libraries, customers, chat memory) and a single-page chat UI.

## Commands

```bash
# Initialize the database (first time only)
python -m trade.database

# Start the server
python server.py                    # default http://127.0.0.1:9119/trade
python server.py --port 8080        # custom port
python server.py --no-browser       # don't auto-open browser

# Install (editable)
pip install -e .
```

This project has **no test suite** and **no linter/formatter configured**. The sole build prerequisite is that `hermes-agent` (from `chefroger/hermes-agent`) must be installed and at a compatible version.

### Key Design Decisions

1. **Hermes Agent is an external dependency** (not vendored). The server starts by checking Hermes version compatibility (>=0.12.0, <0.14.0). If the version is incompatible, it refuses to start. The compatibility matrix is tracked in `COMPATIBILITY.md`.

2. **Session token pattern**: The server generates a random `X-Hermes-Session-Token` on startup and injects it into the served HTML. The SPA uses this for API auth — same pattern as the Hermes dashboard.

3. **Dual chat endpoints**: `/chat` is synchronous (thread pool + 600s timeout); `/chat/stream` uses SSE to emit `tool_start`, `tool_complete`, `thinking`, `response`, `error`, `done` events so the UI can show tool execution progress in real time.

4. **Document libraries = filesystem directories**. Each "library" has a `root_path` pointing to a real directory. The AI agent uses `read_file` / `list_dir` tools to analyze files within. Files can be uploaded via `/libraries/{id}/upload`.

5. **Hindsight is optional**. The `trade/memory.py` module gracefully degrades to no-ops if `hindsight_client` is not installed or configured. Conversation retention to long-term memory is best-effort.

6. **Every module is independently runnable** for smoke testing (`python -m trade.database`, `python -m trade.library`, etc.).

## Hermes Coupling Points

Trade depends on these Hermes internals (watch these on Hermes upgrades):
- `run_agent.AIAgent` — the AI agent class (imported dynamically in chat endpoints)
- `hermes_cli.config.load_config` — reads `~/.hermes/config.yaml`
- `hermes_cli.auth.PROVIDER_REGISTRY` — available LLM providers
- `hermes_cli.env_loader.load_hermes_dotenv` — loads `~/.hermes/.env`
- `hermes_cli.models.name_to_models` — provider-to-models mapping
- `hermes_constants.get_hermes_home` — resolves `~/.hermes` path
- Cognee knowledge graph (tools `cognee_remember` / `cognee_recall` referenced in the system prompt)

The version check in `server.py` (line 33-62) is the first line of defense against breaking Hermes changes.

## Code Annotation Standards

**Every function must have a docstring in Chinese.** This is non-negotiable.

```python
def get_library(library_id: int, company_id: int) -> dict | None:
    """根据 ID 获取指定公司的文档库。

    Args:
        library_id: 文档库主键 ID
        company_id: 公司主键 ID（用于数据隔离校验）

    Returns:
        文档库字典；不存在或不属于该公司时返回 None。

    Raises:
        不会抛异常，所有错误情况均返回 None。
    """
```

**Every if-branch must have a comment** explaining the business logic:

```python
if not agent_identity:
    # 无公司身份配置时，使用通用系统提示词（降级策略）
    system_parts = [TRADE_SYSTEM_PROMPT]
else:
    # 注入公司身份（公司名称 + 产品范围 + 销售风格）
    system_parts = [TRADE_SYSTEM_PROMPT, "", "## 公司身份配置", agent_identity]
```

**Complex list/dict comprehensions must be split with comments**:

```python
active_items = [x for x in items if x.get("active")]      # Step 1: 筛选激活状态
company_items = [x for x in active_items                 # Step 2: 按公司隔离
                 if x.get("company_id") == cid]
```

**Sections of code must be separated by banner comments**:

```python
# =====================================================================
# 第一部分：健康检查（Provider 检查 → 模型配置检查）
# =====================================================================
```

**Annotation coverage requirements**:

| Code type | Minimum requirement |
|-----------|-------------------|
| Function docstring | All functions (including private `_func`) |
| if/else branches | Every branch has a comment |
| Complex loops | Inline comment explaining logic |
| try/except | Comment explaining why and what happens on failure |
| API routes | Inline comment for business meaning |
| Key business logic | Comment explaining decision basis and business rules |
