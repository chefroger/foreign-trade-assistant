> 整理自「大宗师」外贸培训资料（2026-01 ~ 2026-02），作为 Foreign Trade Assistant 的领域知识基础。

## 一、外贸公司核心业务模型

外贸公司的业务本质是两个环节的闭环：

```
引流（找客户） ──────────────────────→ 转化（拿订单）
    │                                        │
    ├── B2B 平台（阿里/中国制造/独立站）        ├── 高级销售员思维
    ├── 社媒（FB / Ins / TikTok / YouTube）     ├── 客户深度分析
    ├── 营销平台（领英 LinkedIn）               ├── 定制化开发信
    └── 数据挖掘（海关数据 / 广交会）           └── 跟进序列与话术
```

**核心公式**：
- 引流 = 曝光 × 精准度 × 内容价值
- 转化 = 客户理解深度 × 沟通策略 × 执行力

## 二、引流 — 五大渠道与方法论

### 2.1 B2B 平台（阿里国际站 / 中国制造网 / 独立站）

**诊断维度**：
- 产品标题：关键词覆盖度、搜索匹配度
- 产品描述：结构化程度、技术参数完整性、FAB（特性-优势-利益）
- 图片 / 视频：数量、质量、场景化展示
- 店铺评分：回复率、成交率、买家评价

**优化方向**：
- 对比同行 Top 10 店铺，逐项找差距
- 关键词策略：核心词 + 长尾词 + 场景词
- 描述模板化：产品参数表 → 应用场景 → 公司优势 → FAQ

### 2.2 社媒内容策略（FB / Ins / TikTok / YouTube）

**学习同行的方法**：
1. 收集 3-5 个同行社媒链接发给 Agent
2. Agent 分析：内容类型分布、发布频率、高互动帖子特征、标签策略
3. Agent 结合本公司产品生成差异化方案

**内容矩阵（领英 5 维度营销法）**：

| 内容类型 | 用途 | 频率建议 |
|---------|------|---------|
| Post - 视频 | 产品展示、工厂实拍、客户案例 | 2-3 次/周 |
| Post - 照片 | 证书展示、发货实拍、团队活动 | 2-3 次/周 |
| Post - 文章 | 行业洞察、技术分享、案例深度解析 | 1 次/周 |
| 投票 (Poll) | 行业趋势、客户偏好调研 | 1 次/周 |
| 文档 (Document) | 产品目录、技术白皮书、Buyer's Guide | 1 次/月 |

### 2.3 领英 LinkedIn — B2B 核心战场

#### 个人账号优化（5 个板块）

| 板块 | 长度要求 | 内容要点 |
|------|---------|---------|
| **Headline** | ≤150 字符 | 产品关键词 + 角色 + 核心优势。三种结构：A) 产品关键词+属性+优势 B) Help 目标客户+业务增长 C) 职位+公司+服务内容 |
| **About** | ≤800 字符 | Who we are → What we make → Who we serve → Basic strengths |
| **Skills** | 10-15 个 | 产品词、岗位技能、常用工具词，逗号隔开 |
| **Experience** | ≤600 字符 | 职位 + 工作内容总结 + 简要成果点 |
| **Media** | 标题 ≤50 字符 | 简单写明展示内容与业务关联 |

**Headline 示例**：
- `[Your Product] Expert | [X] Years Experience | [Key Certification]`
- `I help [target客户 type] grow their business with [your value proposition]`
- `Director of Business Development | Helping [industry] Companies with [sourcing solution]`

#### 公司主页设置

- 公司介绍：行业定位 + 核心优势 + 服务范围
- 产品展示：按产品线分类，每类有独立展示页
- 员工邀请策略：引导员工关联公司主页，扩大触达

#### 领英开发客户方法

**搜索客户的技巧**：
- 搜索习惯保留：Lead → 产品 + marketing + no 广告 + no Admin
- 使用 LinkedIn 常见采购类职位关键词大全进行精准搜索
- 亚马逊找前三的链接，丰富自己的内容

**加好友与触达**：
- 不要触碰（每月 50 封限制）
- 跟视频学习提高 SSI 分数
- Add note 指令：把领英改成 WhatsApp 加
- 分类、主动推

**Smart Link 开发客户**：
- 创建产品 / 行业相关的文档供客户下载
- 引导客户画像来下载（Document 名称 + 引导描述）
- 用目标市场同行的产品关键词

### 2.4 海关数据 / 广交会数据挖掘

**海关数据处理流程**：
1. 导入数据（CSV / Excel）
2. 按 HS 编码 / 产品关键词筛选本行业记录
3. 提取采购商信息：公司名、采购频次、数量、季节规律
4. 按采购量 / 频次排序，确定优先级
5. 匹配本公司产品线，给出开发建议

**广交会数据**：
- 按展馆 / 展区筛选同行业参展商
- 交叉比对公司产品线与参展商需求
- 输出目标客户清单 + 开发优先级

## 三、转化 — AI 高级销售员

### 3.1 Top Sales 核心思维

**核心能力要求**：
- 理解 B2B 销售心理学和决策链路
- 掌握外贸全流程：询盘 → 报价 → 谈判 → 成交 → 跟单
- 能根据不同客户类型调整沟通策略

**三种客户类型与策略**：

| 客户类型 | 关注点 | 应对策略 |
|---------|--------|---------|
| 价格敏感型 | 单价、付款条件、最小起订量 | 强调性价比、长期合作成本优势、阶梯报价 |
| 品质导向型 | 认证、品控流程、样品测试 | 展示证书、工厂审核报告、第三方检测 |
| 关系型 | 信任、沟通频次、售后支持 | 保持定期联系、分享行业信息、提供增值建议 |

**常见客户异议与处理**：
- "价格太高" → 拆解成本构成、对比品质差异、计算长期 TCO
- "已有供应商" → 不做直接竞争、提供差异化价值、建议备份供应商
- "暂时不需要" → 保持联系、分享行业信息、等待时机
- "起订量太大" → 灵活 MOQ 方案、拼单建议、样品试单

**AEO (Ask Engine Optimization)**：
理解为比 FAQ 更高级的自问自答。了解采购商真正关心的核心问题，提前准备好答案。例如：
- "我是澳大利亚的采购商，我想要 ODM，起定量是多少？"
- 针对不同目标市场的典型采购场景，预判并回答关键决策问题

### 3.2 Account IQ — 客户深度分析框架

模拟 LinkedIn Sales Navigator 的 Account IQ 功能，三步分析法：

#### Phase 1: 深度情报挖掘
对目标客户进行四维分析：

| 维度 | 分析要点 |
|------|---------|
| **Strategic Priorities**（战略优先级） | 未来 1-3 年最想做成什么？（数字化转型、拓展新市场、碳中和、降本） |
| **Key Challenges**（关键挑战） | 阻碍目标的具体困难？（供应链不稳定、原材料上涨、合规压力、技术瓶颈） |
| **Financial Signals**（财务信号） | 扩张期（招人、建厂）还是收缩期（裁员、砍预算）？ |
| **Vendor Gaps**（潜在痛点） | 现有供应商哪里做得不够好？ |

#### Phase 2: GAP Selling 价值匹配
```
[客户的战略目标/痛点] + [我方的具体优势] = [破局方案]
```
注意：不罗列产品参数，只列能解决客户 Phase 1 痛点的具体价值。

#### Phase 3: 开发内容交付
输出两份可直接使用的内容：

**Cold Email（顾问式开发信）**：
- Subject: `Idea for [Client Company]'s [Strategic Priority]`
- Opening: `"I was analyzing [Client Name] via Account IQ and noticed that your team is heavily focused on [Strategic Priority]..."`（建立专业感）
- Body: `"Given your challenge with [Challenge], we have helped similar companies leverage [Strength] to achieve [Result]."`（价值桥梁）
- CTA: `"Open to a brief chat to benchmark your current approach?"`（低压力的行动呼吁）

**LinkedIn Connection Note**（加好友话术，≤300 字符）：
- `"Hi [Name], saw [Company] is pushing for [Strategy]. We help peers in [Industry] tackle [Challenge] by [My Value]. Worth connecting?"`

### 3.3 客户画像 — 万花筒能力反推法

**核心理念**：不是问"哪些市场最热"，而是问"你在哪些地方最容易赢"。

**四维能力画像**：

| 维度 | 分析要点 | 决定你能打的战场 |
|------|---------|----------------|
| **Product Power**（产品能力） | 品质、稳定性、用户友好度、市场验证程度 | 高端/中端/低端市场 |
| **Delivery Power**（履约能力） | 产地、海外仓布局、交付速度 | 哪些地区可以快速交付 |
| **Brand Reality**（品牌定位） | 不是贴牌小厂还是有自主品牌 | 客户对你的信任基础 |
| **Service Power**（服务能力） | 售后、技术支持、代理合作 | 能签多大的客户 |

**能力 → 市场匹配示例**：
- 有海外仓布局（美国/东南亚/欧洲）→ 天然适合相应市场
- 高品质 + 稳定性 + 用户友好 → 适合对质量要求高的市场，非低端一次性市场

## 四、外贸全流程操作指南

### 4.1 询盘处理

1. 快速识别询盘质量：客户公司规模、采购意向明确度、需求匹配度
2. 回复时效：24 小时内回复
3. 回复内容结构：
   - 感谢询盘 + 确认需求
   - 产品匹配 + 技术参数
   - 报价（FOB / CIF）+ 交期 + 最小起订量
   - 公司资质简述
   - 下一步行动建议

### 4.2 报价策略

- 首次报价留 10-15% 谈判空间
- 阶梯报价：不同数量不同价格
- FOB vs CIF 根据客户偏好选择
- 付款条件：T/T 30% 定金 + 70% 发货前（首次合作）

### 4.3 谈判技巧

- 先听后说：让客户先表达全部需求
- 价值先于价格：先确认产品匹配，再谈价格
- 条件交换：客户要求降价 → 要求增加数量或缩短付款周期
- 底线清晰：知道自己的成交底线，不亏本接单

### 4.4 跟单流程

1. 签订 PI（Proforma Invoice）
2. 收到定金后安排生产
3. 生产过程中定期更新进度
4. 发货前提供装箱单和质检报告
5. 发货后提供物流追踪号
6. 到货后跟进使用情况

## 七、客户开发 — 方法与邮件技巧

### 7.1 寻找国外客户的 43 个方法（摘要）

核心渠道分类：
- **搜索引擎**：Google 高级搜索、Google Maps、Google 图片搜索 + 关键词组合
- **B2B 平台**：Alibaba、Made-in-China、Global Sources、TradeKey 等
- **海关数据**：北美/南美客户采购记录，分析采购周期、数量、供应商
- **社媒平台**：LinkedIn、Facebook、Instagram、Twitter 搜索决策人
- **展会名录**：广交会、行业展会参展商名录
- **Whois 域名查询**：查网站注册时间（推断公司历史）、注册地址变化（判断发展状况）

### 7.2 客户初步调查方法（Basic Search）

发开发信前必做四步调查：
1. **公司网站研究**：主要页面 + 下载 E-catalog，找出有价值的信息点
2. **搜索引擎技巧**：Google + Google Maps 了解客户实力、网络活跃度、B2B 询价记录、上下游客户
3. **海关数据**：研究客户采购习惯、周期频率、数量和现有供应商
4. **Whois + 社媒**：域名注册时间、反链数量、社交平台活跃度

### 7.3 开发信公式

```
高回复率 = Basic Search, Target Right（初步调查，有的放矢）
           + 避开 SPAM 过滤词
           + 少图片少链接
           + 好的标题
           + 行文简洁清晰
           + 有吸引客户的 Key Point
           + 运气 or 时机（权重不小）
```

**关键原则**：
- 开发信不在于多，在于精，在于坚持
- 先做客户分析，再发信。只花一分钟套模板 = 低回复率
- 网站上只有 info/sales 邮箱 → 打电话问到采购负责人邮箱
- 不要在第一封邮件中使用 FREE、OPPORTUNITY、CHEAP、PRICE、DISCOUNT 等词
- 不要使用主动语态过多，保持正式商业口吻

### 7.4 询盘回复模板与策略

**B 类买家（阿里巴巴等平台）回复原则**：
```
赞许（破冰）→ 解决需求 → 成功案例展示 → 推动成交（限时折扣/运费优惠）
```

**三种破冰方法**：

| 方法 | 适用场景 | 示例 |
|------|---------|------|
| 赞美产品眼光 | 客户选了新款/热款 | "You have good insight, this is our hot style" |
| 展示市场占有率 | 知道客户国家 | "Our products have significant market share in your country" |
| 小客户支持承诺 | 创业型小客户 | "We 100% support you to grow the market together, win-win" |

**二次跟进邮件模板**（客户未回复时）：
1. 免费样品支持（1-2pcs new design free every month）
2. 100% 品质保障（质量问题下次订单免费补发）
3. 小 MOQ 支持（可混款混色）
4. 运费折扣（今日下单 DHL 特价）

### 7.5 询盘分析五维度

| 维度 | 分析要点 |
|------|---------|
| 客户专业度 | 是否使用行业术语、是否问关键参数 |
| 需求明确度 | 有没有明确数量、规格、交期 |
| 真实需求 | 客户到底想解决什么问题（不只是要报价） |
| 采购真实性 | 判断是比价还是真实采购意向 |
| 回复习惯 | 客户通常在什么时间回复、用什么方式 |

## 八、商务谈判与价格策略

### 8.1 客户谈判核心公式

```
成交 = 舒适的沟通 × 风险的控制 × 利润的把控
```

### 8.2 客户接待与热场技巧

**到访前准备清单**：
1. 打印客户来往邮件重点 + 报价表
2. 准备客户感兴趣的样品 + 需要带走的样品
3. 准备产品证书和资质文件
4. 了解客户国家风俗习惯、饮食禁忌
5. 指定接待团队：主谈、副谈、后勤、拍照
6. 准备饮料（咖啡/茶/可乐/矿泉水）、小礼物
7. 名片、笔记本、笔、计算器放在会客室
8. 穿正装

**热场技巧**：
- 接车时聊行程紧不紧、酒店是否方便、是否需要帮忙订酒店/接送机
- 不熟悉的领域不要碰（尤其是宗教、政治）
- 对穆斯林客户：提前准备可做礼拜的小空间、注意饮食禁忌
- 去客户公司拜访时，观察桌椅上是否有亲人照片，可适当聊家人

**谈判中关键**：
- 注意仪态端庄，偶尔活跃氛围但注意时机
- 遇到不懂的问题立即请教同事或工厂，不要硬答
- 争取谈判后立刻准备 PI 并让客户签字
- 谈判后请客户和团队合影

### 8.3 价格谈判策略

**差异化报价表**：用专业的对比表格展示不同配置/付款方式对应的价格，而不是给一个死价。

**付款方式与价格联动**：

| 付款方式 | 价格参考 |
|---------|---------|
| 30% 定金 + 70% 尾款 | 价格 A |
| 50% 定金 + 50% 尾款 | 价格 B（优惠） |
| 70% 定金 + 30% 尾款 | 价格 C（更优惠） |

**五种折扣空间**（不要一次性交底）：
1. 数量折扣 — 量大从优
2. 运费折扣 — 合作首单运费补贴
3. 新客户折扣 — 首单优惠
4. 时效折扣 — 限时下单享受
5. 佣金折扣 — 给中间商留利润

**压制客户还价的四个维度**：
- 产品质量差异化（认证、材料、做工）
- 售后服务差异化（质保期比同行长）
- 供应链稳定性（交付准期率）
- 专业度展示（数据、案例、行业报告）

### 8.4 Trade 公司 vs 小型 B 端客户谈判要点

**小型 B 端客户特点**：关注利润空间、怕风险、对产品不够专业。

**应对策略**：
1. 帮客户算利润：报价 → 运费 → 关税 → 目标市场 C 端售价 → 客户利润
2. 运费用海运慢但便宜 vs 空运快的优劣对比
3. 给阶梯折扣：季节折扣 + 数量折扣 + 付款折扣
4. 售后服务差异化：更长质保、免费备品
5. C 端页面展示：帮客户做好产品详情页、差异化视觉、包装建议
6. 展示其他市场成功案例和热卖数据

## 九、外贸全流程操作手册

### 9.1 样品寄送流程

**应该寄样的情况**：
- 经过几轮磋商后，客户需要深入了解产品
- 规模大、行业有名的客户 → 主动寄样

**寄样前确认**：
- 公司名称、地址、电话、邮编、联系人
- 样品规格、型号、数量
- 付款方式（我方付费 / 客户付费 / 到付）

**寄样到付流程（客户提供快递账号）**：
1. 联系快递客服确认账号有效性
2. 准备样品、填写快递单
3. 准备代理报关委托书、形式发票（三份）
4. FedEx 到付需准备 到付保函 + 营业执照复印件盖章

**后期跟踪**：
- 询问样品是否到达
- 争取客户给出具体反馈
- 建立稳定双向沟通，不只看短期订单

### 9.2 客户验厂接待流程

**接待上报**：提前一周上报部门主管，内容包括公司名称、国家、来访时间、目的、来访人员姓名职务。

**接待后**：立刻整理谈判内容、把合影发到客户邮箱、提交简要汇报（来访效果、总结、经验）。

### 9.3 国外拜访与市场调研

**拜访报告框架**：
1. 总括：时间、人员、路线、拜访概况
2. 市场分析：竞争对手、产品价位、包装、营销渠道、采购习惯、市场反馈
3. 拜访中遇到的问题及应对
4. 展会和推广：当地展会、搜索引擎、黄页、B2B、其他推广方式
5. 费用总结 + 整体评价（亮点/暗点）
6. 后续改进建议

### 9.4 外贸业务管理表格体系

**日常运营**：
- 个人业务管理表、工作周报表、日总结、月报表
- 销售数据分析可视化看板

**客户跟进**：
- 客户跟进管理系统、重点客户跟进分析表
- 客户档案报表、客户背景调查表、客户分析表
- 竞争对手分析表

**订单管理**：
- 外贸订单跟踪表、订单收汇利润统计表
- 年度重要工作计划推进表

## 十二、外贸业务员每日工作清单与自动化定时任务

### 12.1 设计思路

外贸业务员的日常工作有明确的时间规律——不同时区的客户在不同时间段活跃，不同性质的营销任务有最佳的完成时间窗口。将这些规律固化为自动化定时任务，让 Hermes Agent 在指定时间触发、自动帮助业务员准备和完成工作，可以大幅提升效率。

**核心机制**：Hermes 内置 cron 系统，支持 `create_job(prompt, schedule)` 创建定时任务。Agent 到时间自动执行 prompt，生成的内容通过 `deliver` 参数推送到对应平台（终端提醒 / Telegram / 本地文件）。

### 12.2 每日工作清单与时间表

```
08:00 ─ 早安简报，查看全球夜间动态
08:30 ─ 今日待跟进客户清单 + 优先级排序
09:00 ─ 欧洲/中东客户跟进（时区窗口）
09:30 ─ 领英内容发布（Post / Article）
10:00 ─ 开发信发送窗口 1（欧洲客户）
11:00 ─ B2B 平台数据检查（阿里/中国制造）
13:00 ─ 北美客户跟进（时区窗口）
14:00 ─ 社媒内容发布（FB / Ins / TikTok）
15:00 ─ 客户背调 / 市场调研
16:00 ─ 开发信发送窗口 2（北美客户）
17:00 ─ 东南亚/澳新客户跟进（时区窗口）
17:30 ─ 每日工作总结 + 数据记录
18:00 ─ 明日计划生成
```

### 12.3 自动化定时任务详细设计

#### Job 1：早安简报（Daily Briefing）

```yaml
名称: trade-morning-briefing
时间: 08:00（工作日）
Prompt: |
  根据昨天的对话记录和客户跟进表，生成今日早安简报：
  1. 昨天与哪些客户有重要互动？结果如何？
  2. 今天需要跟进的客户列表（按优先级排序）
  3. 今日需要处理的紧急事项（报价到期、样品寄出、PI确认等）
  4. 今日的客户时区窗口（哪些时段适合联系哪些地区的客户）
deliver: local  # 输出到本地终端
```

#### Job 2：领英内容发布助手

```yaml
名称: trade-linkedin-post
时间: 09:30（工作日）
Prompt: |
  根据我司产品资料库和行业动态，生成本日领英发布内容：
  1. 今日 Post 建议（话题 + 文案 + 配图建议 + 标签）
  2. 如果今天是周一/周三/周五，额外生成一篇 Article 草稿
  3. 建议今天适合参与的同行帖子（点赞/评论互动）
  输出后请提醒我：可直接复制粘贴到领英发布。
deliver: local
```

#### Job 3：欧洲客户开发信窗口

```yaml
名称: trade-eu-outreach
时间: 10:00（工作日）
Prompt: |
  现在正是欧洲客户上班时间（CET 9:00-10:00）。
  1. 从客户跟进表中筛选近 7 天未联系的欧洲客户
  2. 为每个客户生成一封简短跟进邮件（3-5 句即可，不要推销腔）
  3. 检查是否有需要发送的开发信（从上周海关数据/广交会数据中筛选的欧洲目标客户）
deliver: local
skills: [b2b-document]
```

#### Job 4：北美客户开发信窗口

```yaml
名称: trade-na-outreach
时间: 16:00（工作日）
Prompt: |
  现在正是北美客户上班时间（EST 7:00-8:00，客户开始看邮件）。
  1. 从客户跟进表中筛选近 7 天未联系的北美客户
  2. 为每个客户生成一封简短跟进邮件
  3. 检查是否有需要发送的开发信（上周数据中筛选的北美目标客户）
deliver: local
skills: [b2b-document]
```

#### Job 5：每日工作总结

```yaml
名称: trade-daily-summary
时间: 17:30（工作日）
Prompt: |
  请根据今天的对话记录总结今日工作：
  1. 今天联系了多少客户（按地区分类）
  2. 今天发送了多少开发信/跟进邮件
  3. 今天发布了多少社媒内容
  4. 今天有无新的询盘、报价、样品需求
  5. 今天的成交进展（哪些客户推进了）
  6. 明天需要重点跟进的 3 个客户
  将结果保存到 工作日报.md，追加模式。
deliver: local
```

#### Job 6：周报生成（每周五）

```yaml
名称: trade-weekly-report
时间: 17:30（每周五）
Prompt: |
  汇总本周所有工作日报，生成周报：
  1. 本周数据总览（联系客户数、发开发信数、社媒发帖数、新询盘数、样品数、PI数、成交数）
  2. 本周重点客户进展
  3. 下周工作计划
  4. 本周遇到的问题和需要的支持
deliver: local
```

### 12.4 自动化任务配置说明

项目可通过 cron 系统配置定时任务：
- 早安简报：每个工作日 08:00
- 领英内容：每个工作日 09:30
- 欧洲开发信：每个工作日 10:00
- 北美开发信：每个工作日 16:00
- 每日总结：每个工作日 17:30
- 周报生成：每周五 17:30
